The Challenge

The client had 14 static HTML pages designed with Tailwind CSS that needed to be integrated into their existing WordPress site. The pages included legal plan subscription pages and workflow kit product pages.

  • Header Overlap Issue â€” The existing theme used a fixed header that overlapped with new page content
  • CSS Conflicts â€” Tailwind CSS classes conflicted with the existing Paperstreet theme styles
  • Content Management â€” The client wanted to easily update pricing without touching code
  • Maintainability â€” 14 separate templates would be difficult to maintain long-term

My Approach

2 Templates Instead of 14

Instead of creating 14 individual templates, I designed 2 reusable templates powered by ACF (Advanced Custom Fields). Adding new plans or kits requires zero code changes.

Child Theme Implementation

All customizations were built in a child theme to protect code from parent theme updates and keep modifications organized.

Native CSS with BEM Naming

Converted Tailwind utility classes to native CSS with BEM naming convention to avoid conflicts with the existing theme.

/* Before: Tailwind */ <div class="bg-blue-600 text-white p-8"> /* After: Native CSS with BEM */ <div class="hero-section hero-section--primary">

Single Source of Truth Architecture

Built a system where pricing data is stored once in ACF fields and dynamically pulled into all overview pages. Update once, reflect everywhere.
